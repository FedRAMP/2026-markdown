# FedRAMP Certification Classes

## Security Categories vs. Certification Classes

NIST FIPS-199 provides a standard for categorizing the security of information and information systems at low, moderate, and high security categories. Agencies use this standard to evaluate confidentiality, integrity, and availability security objectives related to agency-specific systems and use cases.

FedRAMP defines certification classes that represent the amount of information available in a FedRAMP certification package. Certification at higher classes means that the cloud service provider has provided and is expected to continue to provide higher fidelity, timely data about their cloud service offering to FedRAMP and their agency customers. Generally, each successive class introduces increasing levels of rigor in demonstrating security capability, as opposed to introducing requirements for additional capabilities.


## Why the distinction?

In order to categorize an information system, the agency must analyze the security categorization (low/mod/high) and the security objectives (confidentiality/availability/integrity) of the information in the system. Use cases of the same cloud service offering could vary widely from agency to agency. FedRAMP does not have insight into how each agency might use a particular cloud service, so categorizing cloud service offerings into security categorizations is not appropriate.

Certification classes may be used by agencies to quickly gauge the general level of investments into security capabilities that each service provider has made, however final agency risk determinations must take into account how the agency will use the cloud service.
Using FedRAMP Certification Classes
Federal information systems could be composed of components that separately process information at different security categorization levels. Agencies are encouraged to properly architect federal information systems to use cloud services that appropriately match the security categorization of the information
